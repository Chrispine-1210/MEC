Mtendere Admin Quick Reference Guide

Version 1.0 · 23 July 2026 · Prepared by Chrispine Mndala / Aöthothe Technologies

Role access at a glance

RolePrimary purposeTypical access
Super AdminOwns the platform, accounts, roles and settings.All admin modules, including Users, Roles & Permissions and Settings.
AdminRuns daily operations and reviews platform activity.Applications, Messages, Subscribers, Communications, Analytics, Activity, Payments and content modules.
Writer / Content ManagerMaintains public content.Scholarships, Job Opportunities, Events, Partners, Blog Posts, Team Members and Media Governance.
ViewerReads high-level operational status only.Dashboard only.

Daily operating flow

  1. Sign in.
  2. Review Dashboard.
  3. Check Applications and Messages.
  4. Handle Subscribers/Communications alerts.
  5. Update urgent Scholarships, Jobs, Events or Blog records.
  6. Record decisions and escalate technical issues.
Security: MFA is currently disabled for administrator handoff. Re-enable after the next authentication update. Never share accounts or expose credentials, MFA secrets, applicant data or payment information.

Top troubleshooting

IssueDo this first
Access deniedCheck assigned role against the manual role matrix.
Email confirmation/unsubscribe link failsStop sending campaigns; verify public base URL, token route and deployed action page.
Save failsCheck required fields, refresh, sign in again, then escalate with sanitized screenshot and timestamp.
Payment issueDo not retry blindly; escalate with transaction reference and timestamp.

Escalation details to capture

Page/module, account role, exact action, timestamp, browser, exact error message and screenshot with sensitive information hidden.
